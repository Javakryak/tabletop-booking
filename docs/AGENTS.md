# docs/AGENTS.md

Rules for project documentation.

## General Rules

- Keep docs consistent with implemented behavior.
- Do not describe unimplemented features as complete.
- Developer docs may be English or Russian, but each file should be internally consistent.
- Do not include secrets, real personal data, production logs, database dumps, or private credentials.

## ADRs

Use `docs/adr` for important architectural decisions, including stack choices, auth strategy, booking conflict strategy, privacy/legal decisions, and deployment architecture.

## Codex Workflow Docs

Use `docs/codex` for prompt logs, dev diary, review checklist, and agent reference material.
